# 최종본이 바뀌었습니다

NHN GAME X AI HACKATHON을 위해 제작 중인 오피스 로맨틱 코미디 추리 게임입니다.

플레이어는 정직원 전환을 앞둔 계약직 신입사원 **한도윤**이 되어, 5일 동안 최종 PT를 준비합니다. 발표 자료가 알 수 없는 이유로 뒤엉키기 시작하고, 플레이어는 사수 **서하린**과 함께 사건의 원인을 추리하면서 업무 성과와 관계를 모두 지켜야 합니다.

## 게임 개요

- 장르: 오피스 럽코 · 추리 · 비주얼 노벨 · 미니게임
- 플레이 기간: 게임 내 5일
- 핵심 목표: 정직원 전환 PT 성공 및 서하린과의 신뢰 형성
- 주요 능력치: 업무력, 호감도, 신뢰도, 체력, 단서
- 엔딩: 해피 엔딩, 동료 엔딩, 배드 엔딩

## 이야기

정직원 전환까지 남은 시간은 단 5일.

한도윤은 최종 PT 과제를 받고 서하린과 함께 발표를 준비한다. 하지만 발표 자료의 문장이 예전 버전으로 돌아가고, AI 자동 요약으로 의미가 달라지며, 최종본에는 서하린의 수정 기록까지 남는다.

플레이어는 회사 업무를 처리하고 메신저로 동료들과 대화하며 단서를 모아야 한다. 상대를 무조건 믿거나 의심하는 것이 아니라, 확인하면서도 몰아붙이지 않는 선택이 좋은 결말로 이어진다.

## 5일간의 진행

| 날짜 | 주요 사건 | 미니게임 |
| --- | --- | --- |
| DAY 1 | 정직원 전환 과제 공개 | 팀원 취향에 맞춰 커피 타기 |
| DAY 2 | 서하린과 함께 자료 점검 | 메신저 업무 분류하기 |
| DAY 3 | 자료에서 첫 이상 징후 발견 | 부장님 몰래 서하린에게 답장하기 |
| DAY 4 | 최종본 변조 오해 발생 | 단서 조사 및 사건 연결하기 |
| DAY 5 | 발표 직전 최종본 복구 | 긴급 PT 준비하기 |

## 현재 구현 범위

현재 버전에는 게임의 첫 타이틀 화면이 구현되어 있습니다.

- 회사 오피스 배경
- 게임 제목 및 스토리 시간 표시
- 새 게임, 이어하기, 기록, 설정 메뉴
- 마우스 및 키보드 메뉴 조작
- 새 게임 선택 시 DAY 1 전환 연출
- 이어하기 선택 시 수동 저장 슬롯 5개 표시
- 브라우저 저장 데이터의 컷씬, DAY, 최근 대사, 저장 시간 표시 준비
- 데스크톱 및 작은 화면 대응

## 실행 방법

별도의 설치나 서버 실행은 필요하지 않습니다.

1. 프로젝트 폴더를 엽니다.
2. `index.html`을 더블 클릭합니다.
3. 웹 브라우저에서 게임 타이틀 화면이 실행됩니다.

## 조작 방법

| 입력 | 동작 |
| --- | --- |
| 마우스 이동 | 메뉴 선택 |
| 마우스 클릭 | 선택한 메뉴 실행 |
| 방향키 | 이전 또는 다음 메뉴로 이동 |
| Enter | 선택한 메뉴 실행 |

## 프로젝트 구조

```text
NAN_GAME_TITLE/
├─ index.html                    # 타이틀 화면 구조
├─ css/
│  └─ title-screen.css           # 디자인 및 반응형 스타일
├─ js/
│  └─ title-screen.js            # 메뉴 조작과 화면 연출
├─ assets/
│  └─ office-background.png      # 회사 오피스 배경
└─ README.md                     # 프로젝트 안내
```

## 주요 등장인물

### 한도윤

정직원 전환을 앞둔 계약직 신입사원. 성실하고 책임감이 강하지만 모든 일을 혼자 해결하려는 버릇이 있다.

### 서하린

한도윤의 사수이자 추리 파트너. 일할 때는 냉정하고 차분하지만 가까워지면 장난스럽고 다정한 모습을 보여준다.

### 강민재

한도윤과 함께 정직원 전환 심사를 받는 동기. 경쟁자처럼 보이지만 악역은 아니며, 잘못된 도움으로 사건의 오해를 키운다.

### 박태식

팀의 부장. 중요한 지시를 애매하게 전달하는 바람에 발표 자료가 뒤엉키는 원인을 제공한다.

### 윤세아

회사 시스템과 프린터 기록을 관리하는 지원팀 직원. 조사 과정에서 중요한 단서를 제공한다.

### 나나봇

회사 내부 문서 작성 보조 AI. 자료를 보기 좋게 수정하려다가 핵심 의미를 바꾸며 사건의 기술적 원인이 된다.
