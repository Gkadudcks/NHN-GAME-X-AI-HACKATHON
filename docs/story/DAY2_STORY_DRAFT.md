## DAY 2의 역할

DAY 2는 범인을 찾는 날이 아니라, 나중에 무엇이 바뀌었는지 증명할 **두 번째 기준점**을 만드는 날이다.

플레이어는 하린과 함께 DAY 1 초안의 수치를 검증하고 `v0.2_working`을 만든다. 이 과정에서 과거 폴더에 남은 자동 동기화 규칙과 소유자 `서하린`을 발견한다. 이름만 보면 하린이 수상하지만, 마지막 실행 시각은 오래전이고 규칙은 현재 비활성 상태다. 플레이어는 여기서 처음으로 **자동화 소유자와 실제 실행자는 다를 수 있다**는 원칙을 배운다.

저녁에는 하린이 과거 사건의 전모를 말하지 않은 채 “예전에 파일 문제로 사람까지 잃은 적이 있다”는 정도만 드러낸다. 도윤은 하린을 무조건 믿거나 즉시 의심하는 대신, 기록을 함께 확인하는 태도를 선택할 수 있다.

## 먼저 고정할 중요한 포인트

### 이후 날짜를 위해 유지해야 하는 사실

1. **[고정] DAY 2에는 현재 PT 파일의 변조가 발생하지 않는다.**
   - DAY 3의 첫 변조가 약해지지 않도록 DAY 2 종료 시점의 `v0.1`과 `v0.2_working`은 모두 정상이어야 한다.
2. **[고정] 서하린은 과거 자동화의 소유자다.**
   - 소유자 표시는 숨기지 않는다. 플레이어가 직접 보고 단서로 기록한다.
3. **[고정] 마지막 실행 시각은 오래전이다.**
   - 하린을 의심하게 만드는 정보와 하린이 현재 실행하지 않았다는 반증을 같은 날 제공한다.
4. **[고정] 실제 실행 계정과 기기 정보는 DAY 4까지 공개하지 않는다.**
   - DAY 2에는 “실행 기록은 소유자 정보와 별도로 남는다”는 시스템 원칙만 알려준다.
5. **[고정] 강민재는 악의적인 방해자가 아니다.**
   - 구버전 자료 재활용을 권하지만, 아직 자동화를 켰다는 사실은 보여주지 않는다.
6. **[고정] 하린은 도윤의 원본과 작업 기록을 꼼꼼히 지킨다.**
   - 과거를 피하는 행동과 현재의 책임감이 동시에 보여야 한다.
7. **[고정] 스탯은 업무력·호감도·신뢰도만 사용한다.**
   - 체력 스탯은 사용하지 않는다.
8. **[고정] 오늘의 업무 정산을 마친 뒤 퇴근한다.**
   - DAY 2는 야근 자체가 업무 파트이므로 정산은 야근 장면 뒤에 배치한다.

### 수정 가능성이 높은 항목

1. **[수정 후보] 검증 데이터의 정확한 숫자**
   - 현재 가안: 최근 7일 신규 설치 12,480명, 첫 전투 도달 8,502명, 첫 전투 이전 이탈 31.9%.
   - 이후 PT 복구 미니게임에도 재사용할 숫자이므로 DAY 3~5 구현 전 확정해야 한다.
2. **[수정 후보] 자동화 명칭과 과거 실행 시각**
   - 현재 가안: `온보딩_공용슬라이드_동기화`, 마지막 실행 `2024-11-07 23:48`.
   - DAY 4 로그 조사 화면과 동일한 이름·시각을 써야 한다.
3. **[수정 후보] 메신저 분류 카테고리**
   - 초기안의 `부장님 / 히로인 / 업무 / 잡담 / 스팸`은 사람과 내용 기준이 섞여 있어 정답이 애매해질 수 있다.
   - 본 초안은 `부장 지시 / 하린 협업 / PT 자료 / 사내 잡담 / 외부 스팸`으로 문구를 구체화한다.
4. **[수정 후보] 하린이 과거를 어디까지 말하는지**
   - DAY 2에는 “과거 파일 사고가 있었다”까지만 밝힌다.
   - 혼자 책임졌던 일, 원본 소실, 동료의 퇴사는 DAY 4 고백에 남긴다.
5. **[수정 후보] DAY 2 이벤트 CG 사용 여부**
   - 야근 장면은 신규 야간 사무실 배경만으로도 성립한다. CG는 감정선 테스트 후 선택한다.

## 메신저 미니게임 상세 가안

### 기본 규칙 **[수정 후보]**

- 연습 3개 + 본게임 15개 메시지
- 제한 시간 45초
- 메시지는 한 장씩 나타나며 클릭 또는 드래그로 분류
- 핵심 메시지 3개는 오분류 시 감점이 크다.
- 하린 메시지를 모두 정확히 분류하면 호감도 보너스가 붙는다.
- 박태식 메시지를 놓치면 오후에 하린이 재전달하며 업무력 보너스를 잃는다.
- 스팸 오분류는 작은 감점만 주어 초반 난이도를 완충한다.

### 이야기와 연결되는 실패

실패는 즉시 게임오버가 아니라 다음 장면의 대사와 평가를 바꾼다.

- 부장 지시 누락: 하린이 “구버전 숫자는 참고만 하라는 메시지부터 다시 봐요.”라고 지적한다.
- 하린 협업 누락: 하린이 `v0.2_working`을 직접 만들어 두고 도윤의 신뢰도가 오르지 않는다.
- PT 자료 누락: 분석 시트 확인이 늦어져 야근 시간이 10~20분 늘어난다.
- 잡담/스팸 오분류: 민재의 가벼운 놀림 대사만 추가한다.

   - 안정 ID 가안: `background.office.night`

2. **[이미지 선택] 야근 이벤트 CG**
   - 안정 ID 가안: `event_cg.day2.harin_overtime_monitor`
   - 도윤 시점에서 하린이 모니터 옆에 캔 커피를 내려놓는 장면.
   - 화면에 읽을 수 있는 문서 텍스트나 로고를 넣지 않는다.
   - 일정이 부족하면 야간 배경 + 기존 하린 스탠딩 + 컵 UI 소품으로 대체한다.
3. **[이미지 선택] 하린 컵 소지 스프라이트**
   - 허용된 `holding_cup` pose와 `gentle_smile` expression 조합을 우선한다.
   - CG를 만들 경우 별도 스프라이트는 생략할 수 있다.

**[구현 주의]** 새 이미지를 만들기 전에 현재 런타임 에셋과 `assets/art/manifests/art-assets.json`의 캐릭터 ID 불일치를 정리해야 한다. 신규 에셋은 manifest에 계획 버전을 먼저 등록하고, 승인본을 덮어쓰지 않으며, 생성 로그와 검증을 완료한 뒤 안정 ID로 참조한다.
