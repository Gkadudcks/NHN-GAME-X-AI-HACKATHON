1. 메신저 대화방 이름은 `박태식 부장`, `서하린 사수`, `강민재 동기`, `윤세아`, `PT 전환과제 TF`처럼 **사람 또는 실제 채팅방 이름**을 유지한다.
2. 단서는 메신저 분류명이 아니라 별도의 단서 탭에 기록한다.
3. 배경이나 캐릭터 이미지가 없으면 제작을 기다리지 않고 **직사각형 플레이스홀더 + 장면 설명 텍스트**로 대체한다.
4. DAY 2가 이어받아 대사와 결과에 사용하는 정보는 다음 세 가지다.
   - DAY 1 종료 시점의 업무력·호감도·신뢰도 수치
   - DAY 1 나나봇 사용 방식
   - DAY 1 편의점 선택
5. 기존 단서 목록은 단서 탭에 계속 남지만, 모든 DAY 1 선택을 DAY 2 분기로 만들지는 않는다.
6. 수치·버전·백업 이야기는 필요한 장면에만 짧게 사용한다.
7. 자동화 기록은 별도 조사 파트로 다루지 않고, 다른 업무 도중 우연히 보고 지나간다.
8. 로컬 백업은 사용하지 않는다. 사내 클라우드의 변경 이력, 권한, 복원 지점을 이용한다.
9. 메신저 미니게임은 순서대로 문제를 푸는 방식이 아니라, 상단 업무 테이블에 요청이 불규칙하게 나타나는 실시간 대응 방식으로 다시 검토한다.

## DAY 2의 새 리듬

DAY 2의 메인 감정은 “의심”보다 **같이 일해 보니 서로의 예상 밖 모습을 알게 되는 날**에 가깝다.

```text
어제의 편의점 회수
→ 짧은 수치 검증
→ 플레이어가 하위 업무 선택
→ 여러 종류의 사내 요청이 몰리는 미니게임
→ 민재와 업무 밖 점심 대화
→ 선택한 하위 업무 진행
→ 자료를 찾다가 과거 자동화를 우연히 목격
→ 가벼운 공동 테스트와 야근
→ 하린의 과거를 한 문장만 엿봄
```

수치 검증과 자동화 기록은 추리의 뼈대지만, 플레이 시간 대부분을 차지하지 않는다. DAY 2의 플레이 감각은 “회사에서 여러 일을 동시에 처리하다가 하린과 조금 가까워졌고, 지나가듯 이상한 기록 하나를 봤다”가 되어야 한다.

## DAY 2 미니게임 아이디어 조사

### A. 업무 알림 쳐내기 · 추천

상단의 넓은 업무 테이블에 메시지와 요청 카드가 불규칙한 위치와 순서로 나타난다. 플레이어는 카드가 사라지기 전에 클릭하고, 내용에 맞는 빠른 행동을 선택한다.

- 요청 카드는 왼쪽부터 차례로 나오지 않는다.
- 같은 시점에 최대 3개까지 겹쳐 나타난다.
- 새 요청이 들어오는 동안 이전 요청을 처리해야 한다.
- 각 카드에는 실제 발신자 이름과 프로필이 표시된다.
- 처리 결과는 해당 인물의 실제 메신저 대화방에 남는다.
- 추리 단서는 별도 단서 탭에 기록된다.

빠른 행동 예시:

- `답장 보내기`
- `파일 전달`
- `일정 등록`
- `담당자에게 넘기기`
- `나중에 확인`
- `스팸 차단`

이 방식은 분류함 이름을 억지로 만들지 않아도 되고, “요청이 슉슉 나온다”는 원하는 감각을 가장 직접적으로 살릴 수 있다.

### B. 데스크톱 창 전환

메신저, 일정표, 사내 위키, 파일 요청 창이 번갈아 떠오르고 올바른 창을 빠르게 찾아 처리한다.

- 장점: 실제 사무 업무 느낌이 강하다.
- 단점: 모바일이나 작은 화면에서 조작이 복잡하다.
- DAY 5 복합 업무와 비슷해질 위험이 있다.

### C. 우선순위 스택

요청 카드가 계속 쌓이고 플레이어가 긴급도에 따라 위아래 순서를 바꾼다.

- 장점: 업무 우선순위 판단을 명확히 보여준다.
- 단점: 실시간으로 즉시 대응하는 손맛이 약하다.
- “왜 이것이 더 긴급한가”에 대한 텍스트 설명이 많아질 수 있다.

### D. 사내 요청 연결하기

들어온 요청을 박태식, 서하린, 강민재, 윤세아 중 맞는 담당자에게 전달한다.

- 장점: 등장인물 역할을 자연스럽게 학습한다.
- 단점: DAY 4 카드 연결과 시각적 구조가 비슷하다.

### E. 자료 오류 찾기

표나 문장 속 다른 숫자, 누락된 출처를 제한 시간 안에 찾는다.

- 장점: PT 스토리와 직접 연결된다.
- 단점: DAY 2 전체가 다시 숫자와 검증 이야기로 몰린다.
- 이번 수정 방향에는 부적합하다.

### 현재 추천 결론

`A. 업무 알림 쳐내기`를 프로토타입 대상으로 삼는다. 다만 정답 행동, 카드 유지 시간, 동시 등장 개수는 실제 조작 테스트 후 확정한다.

## 미니게임 상세 가안 · 업무 알림 쳐내기

### 화면

- 위쪽 70%: 업무 테이블. 요청 카드가 랜덤한 위치에 나타난다.
- 오른쪽: 기존 메신저 목록. 대화방 이름은 사람 이름과 실제 그룹명으로 유지한다.
- 아래쪽: 현재 잡은 카드에 사용할 수 있는 2~3개의 행동 버튼.
- 카드에는 `발신자`, `요청`, `남은 시간`만 짧게 표시한다.

### 진행

1. 3개의 연습 요청으로 행동 아이콘을 익힌다.
2. 본게임에서는 40~50초 동안 14~18개의 요청이 불규칙하게 등장한다.
3. 카드를 클릭하면 시간이 잠깐 느려지지만 완전히 멈추지는 않는다.
4. 행동을 고르는 동안에도 다른 요청의 남은 시간이 줄어든다.
5. 긴급 요청을 놓치면 업무력 감점, 하린의 개인 요청을 놓치면 관계 보너스를 받지 못한다.
6. 핵심 스토리 정보는 놓쳐도 나중에 다시 확인할 수 있다.

### 요청 구성

PT 관련 요청은 전체의 3분의 1 이하로 제한한다.

| 발신자 | 요청 예시 | 행동 예시 | 성격 |
| --- | --- | --- | --- |
| 박태식 부장 | 11시 회의 시간을 일정에 넣어 줘. | 일정 등록 | 긴급 업무 |
| 서하린 사수 | 아침에 뽑은 원본 통계 링크만 보내주세요. | 파일 전달 | PT 업무 |
| 윤세아 | 오후 빌드 점검 참여 가능 여부 알려주세요. | 답장 보내기 | 다른 프로젝트 업무 |
| 강민재 동기 | 구내식당 줄 너무 긴데 밖에서 먹을래? | 나중에 확인 | 일상 |
| 운영팀 | 이벤트 공지 첫 문장만 오탈자 확인 부탁드립니다. | 담당자에게 넘기기 또는 처리 | 다른 프로젝트 업무 |
| 인사팀 | 이번 달 보안 교육 미수료 안내입니다. | 일정 등록 | 사내 업무 |
| 외부 발신 | 무료 PPT 템플릿 계정 인증 요청 | 스팸 차단 | 함정 |

### DAY 2 하위 업무와 연결

오전에 선택한 하위 업무에 따라 카드 3개가 달라진다.

- 타회사 온보딩 조사 선택: 경쟁 게임 캡처 요청, 비교표 링크, 앱 설치 알림
- 유저 리뷰 태깅 선택: 고객센터 리뷰 묶음, 감정 태그 요청, 중복 리뷰 확인
- 첫 10분 흐름 정리 선택: 플레이테스트 참여 요청, 화면 녹화 링크, 동선 메모

### 결과

- 높은 점수: 업무력 `+2`, 신뢰도 `+1`, 하린 요청을 모두 처리했다면 호감도 `+1`
- 보통 점수: 업무력 `+1`
- 낮은 점수: 업무력 `-1`, 야근 대사와 종료 시각이 조금 달라짐

미니게임 실패로 단서를 영구히 잃거나 DAY 3 진입이 막히지는 않는다.
